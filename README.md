# 📋 Custom Search Plugin for Obsidian

**custom-search** 是一個 Obsidian 搜索增強插件，無需建立索引(原生索引會引起延遲)，即時搜索，適合需要在大規模筆記庫（數千至數萬文件）中進行精準、高效搜索的用戶。它支持**正則表達式**、**文件範圍自定義**、**文件組/組合管理**、**結果排序**以及**多種結果展示模式**等功能，並將搜索結果顯示在**右側邊欄**，方便快速瀏覽和跳轉。

此外，插件還提供了**手寫搜索查詢**功能，支持完整的 Obsidian 原生搜索語法，通過**輔助編輯器**進行可視化構建查詢。

## 🚀 快速開始

### 安裝

1. 下載本項目的 `main.js` 和 `manifest.json` 文件。
2. 在 Obsidian 庫的 `.obsidian/plugins/` 目錄下，創建一個名為 `custom-search` 的文件夾。
3. 將下載的兩個文件放入 `custom-search` 文件夾中。
4. 在 Obsidian 中，進入 `設置` → `第三方插件`，**禁用安全模式**。
5. 在已安裝插件列表中，啟用 `Custom Search` 插件。

### 基本使用流程

1. **選中文本** — 在編輯器中選中要搜索的關鍵詞
2. **執行命令** — 使用命令或右鍵菜單，選擇 `預設範圍快搜` 或 `搜索面板`
3. **選擇模式** — 在彈出的對話框中選擇搜索範圍和方式
4. **查看結果** — 結果顯示在右側邊欄
5. **點擊跳轉** — 點擊任意結果跳轉到對應位置（按住 `Ctrl` 可在新標籤頁打開）

## 🔍 搜索方式

### 方式一：預定範圍搜索

使用預設的文件模式進行快速搜索。

- 「初始默認預定範圍」可在設置中配置
- 可將文件組或組合設為「臨時默認預定範圍」，設置後「預定範圍」按鈕會顯示 ⭐ 標記
- 適用於經常在固定範圍內搜索的場景
- 點擊「預定(原生)」按鈕可轉為 Obsidian 原生搜索

### 方式二：自定義範圍搜索

手動指定文件名匹配模式，在匹配的文件中搜索內容。

- **文件名匹配**：單行輸入框支持多行模式（點擊後自動展開），每行一個正則表達式，支持包含/排除規則
- **內容搜索**：支持普通文本或正則表達式（自動檢測 `+*?|()[]{}` 字符）
- **排除規則**：以 `!` 開頭的行表示排除匹配該模式的文件（支持轉義 `\!` 匹配字面感嘆號）
- **修改版標記**：如果應用了文件組/組合後又修改了編輯區內容，按鈕會顯示「已修改」標記
- 點擊「自定義(原生)」按鈕可轉為 Obsidian 原生搜索

#### 包含/排除規則示例

```
# 包含規則（正常匹配）
^(集異門|法蘊|品類|識身)足論\.md
(婆沙|正理|本義抄)\d+-\d+\.md

# 排除規則（以 ! 開頭）
!.+疏22\.md
!^vibhasa\/others\/
!.*甘露味.*\.md
```

插件支持四種搜索模式：
- **包含模式** (`include-only`)：只匹配包含規則的文件
- **排除模式** (`exclude-only`)：從所有文件中排除匹配排除規則的文件
- **排除優先** (`exclude-first`)：先排除，再選取包含
- **包含優先** (`exclude-last`)：先選取包含，再從中排除


包含/排除模式自動檢測邏輯

| 模式 | 條件 | 行為 |
|:---|:---|:---|
| `include-only` | 只有包含規則 | 只匹配包含規則的文件 |
| `exclude-only` | 只有排除規則 | 從所有文件中排除匹配排除規則的文件 |
| `exclude-first` | 有包含規則，且第一個包含行之前有排除行 | 先排除，再選取包含 |
| `exclude-last` | 有包含規則，且第一個包含行之後有排除行 | 先選取包含，再從中排除 |


### 方式三：原生搜索查詢

- 前兩種方式均可自動轉換成**原生搜索**，填入 Obsidian 搜索框並自動執行搜索。
- **手寫模式**：直接編寫 Obsidian 原生搜索語法，利用 Obsidian 內置的搜索功能。
- **輔助編輯器**：可視化構建查詢，無需記憶複雜語法。

原生搜索適用場景：
- 使用複雜的搜索條件組合（AND/OR/NOT）
- 搜索特定類型的內容（任務、區塊、章節等）

### 三種方式對比

| 方式 | 優點 | 適用場景 |
|:---|:---|:---|
| 預定範圍 | 快速、配置固定 | 日常頻繁使用的搜索範圍 |
| 自定義範圍 | 靈活、精準控制 | 臨時需要調整搜索範圍 |
| 原生搜索 | 功能最強大 | 複雜條件組合搜索 |

## 📁 管理文件組/組合

### 文件組 (Group)

由多個正則表達式模式組成的集合，支持包含/排除規則。

```
# 示例：俱舍論相關文件組
(俱舍論|AKBh)\d+\.md
^kosa\/ju\/
俱舍論記疏\d+\.md
!.+疏22\.md          # 排除包含「疏22」的文件
```

操作：
- 創建/刪除文件組
- 編輯組內正則模式（每行一個，支持註釋 `#`，支持排除規則 `!`）
- 點擊組標題可折疊/展開

### 組合 (Combination)

將多個文件組組合成一個更大的範圍。

操作：
- 創建/刪除組合
- 從文件組列表中選擇添加到組合
- 已加入的組以標籤形式顯示，支持單獨移除

### 應用選項

| 選項 | 功能 |
|:---|:---|
| ⭐ 設為臨時默認預定範圍 | 將選中的文件組/組合作為臨時默認搜索範圍 |
| 🗑️ 清除臨時默認 | 清除已保存的臨時默認預定範圍，恢復初始設置 |
| 🔍 應用為自定義範圍（本次搜索） | 將模式填充到主窗口編輯區，並顯示「📌」標記（可繼續微調） |
| ✍️ 寫入手寫搜索查詢區 | 生成 Obsidian 原生搜索查詢語法，自動填入「手寫搜索查詢」對話框 |

⭐將任意文件組或組合設為臨時默認預定範圍後：
- 「快速預設範圍搜索」命令直接使用此範圍
- 「搜索面板」中的「預定範圍」選項使用此範圍，並顯示 ⭐ 標記
- 點擊「清除臨時默認」可恢復使用「初始默認預定範圍」

#### ✍️ 手寫搜索查詢對話框

提供兩種編輯模式：

##### 手動編輯模式

直接編寫 Obsidian 原生搜索語法。

支持的語法示例：
```
line:(佛即知子曹疑) file:/義足經/
line:/為顯自師[，\.]?其體尊高.*故先讚德/ path:^kosa/ju/
(tag:#哲學) (file:思考筆記.md)
```

##### 輔助編輯模式

通過表單可視化構建搜索查詢：

- **搜索類型**：all / file / path / tag / content / line / block / section / task / task-todo / task-done
- **搜索內容**：輸入關鍵詞或正則表達式
- **邏輯運算符**：AND / OR / NOT（第一行無運算符）
- **修飾符**：
  - 🔤 大小寫敏感（match-case）
  - ◌̈ 區分變音符號（diacritics）
  - .* 正則表達式（regex）
- **快捷填充**：點擊類型旁的圖標 📁，可從 vault 中選擇文件/標籤/路徑快速填入
- **生成查詢**：點擊「✍️ 轉入編輯」將生成的查詢語法填入手動編輯區
- **重置表單**：清空所有條件，恢復初始狀態

### 預定範圍標籤頁

設置插件的「初始默認預定範圍」文件模式。這是「快速預設範圍搜索」和「預定範圍」按鈕的基礎範圍。

## 🖱️ 結果面板

### 基本操作

| 操作 | 效果 |
|:---|:---|
| 點擊結果行 | 跳轉到對應文件的行，並高亮選中關鍵詞 |
| `Ctrl` + 點擊 | 在新標籤頁打開 |
| 點擊左側序號 | 循環切換該條目顯示模式（A→B→C→A） |
| 全局 A/B/C 按鈕 | 切換所有結果的默認顯示模式 |
| Top 按鈕 | 滾動到面板頂部 |
| 重新搜索按鈕 | 使用當前搜索方式重新打開面板（方便修改搜索方式） |

### ⌨️ 鍵盤導航

在結果面板獲得焦點時，可以使用以下快捷鍵：

| 快捷鍵                 | 功能            |
| :------------------ | :------------ |
| `↑` / `↓`           | 選擇上一個/下一個搜索結果（支持循環） |
| `Enter`             | 跳轉到當前選中的結果    |
| `Home` / `End`      | 跳轉到第一個/最後一個結果 |
| `Escape`            | 清除當前選中結果的焦點，將焦點轉移到面板容器   |
| `Ctrl+←` / `Ctrl+→` | 切換上一條/下一條搜索歷史 |
| `Ctrl+\`            | 顯示搜索歷史列表（快捷鍵方式） |

> 💡 提示：點擊結果面板任意位置後即可使用鍵盤導航；按 Escape 可將焦點轉移到面板容器。

### 📊 搜索結果顯示（三種模式）

| 模式 | 顯示方式 | 適用場景 |
|:---|:---|:---|
| **A (單行)** | 只顯示匹配行，動態截取適應面板寬度 | 快速瀏覽，節省空間 |
| **B (3行+滾動)** | 顯示匹配行及其前後各一行，固定高度 | 需要上下文參考 |
| **C (展開)** | 顯示匹配行及其前後各一行，完全展開 | 需要完整上下文 |

| 切換操作                 | 效果                           |
| :------------------- | :--------------------------- |
| 點擊頂部 A/B/C 按鈕        | 全局切換，一次性設置所有結果的默認顯示模式        |
| 懸浮文件頭「«」按鈕 → 選 A/B/C | 批量設置當前文件下所有結果的模式 |
| 點擊每個結果左側的序號          | 獨立切換該條目的模式（循環：A → B → C → A） |

### 📜 搜索歷史

插件會自動保存每次主動搜索的記錄，方便回溯和重複使用。

#### 圖形化操作

| 操作           | 說明            |
| :----------- | :------------ |
| 點擊 `❮` / `❯` | 切換上一條/下一條搜索歷史 |
| 長按 `❮` / `❯` | 顯示歷史列表面板（長按 300ms） |
| 點擊歷史列表中的條目   | 加載該歷史並重新搜索    |
| 點擊歷史列表中的 `✕` | 刪除該條歷史記錄      |
| 點擊「清除所有歷史」   | 清空全部搜索歷史      |
| 鍵盤導航（歷史面板內） | `↑`/`↓` 選擇，`Enter` 加載，`Escape` 關閉 |

#### 快捷鍵操作

| 快捷鍵       | 功能          |
| :-------- | :---------- |
| `Ctrl+←`  | 上一條搜索歷史（更舊） |
| `Ctrl+→`  | 下一條搜索歷史（更新） |
| `Ctrl+\`  | 顯示歷史列表面板 |

#### 歷史面板功能

- 半透明毛玻璃效果背景
- 顯示當前位置（如 `(2/10)`）
- 當前激活條目高亮顯示
- 支持鍵盤導航（`↑`/`↓` 移動，`Enter` 加載）
- 鼠標懸停時自動退出鍵盤導航模式

## ⚙️ 設置說明

打開設置 → custom-search 插件，共有 5 個選項卡：

### 1. 搜索設置

| 設置項 | 說明 |
|:---|:---|
| 默認顯示模式 | A（單行）或 B（3行+滾動）或 C（展開） |

**默認搜索範圍（初始預定範圍）**
設置「快速預設範圍搜索」使用的初始預定文件名模式，每行一個正則表達式。支持註釋行（`#` 開頭）。

### 2. 優先級設置

啟用排序後，搜索結果按以下優先級排列：

| 優先級 | 名稱 | 說明 |
|:---|:---|:---|
| 1 | 文件優先級 | 匹配正則的文件/路徑優先顯示（按行順序） |
| 2 | 文件組間優先級 | 拖拽調整組的顯示順序（越靠上越優先） |
| 3 | 文件組內模式優先級 | 調整組內正則行的匹配順序（越靠上越優先） |
| 4 | 默認順序 | 行號、文件名等 |

#### 文件優先級示例

```
# 優先顯示這些路徑/文件
^kosa\/ju\/
^重要/
核心概念\.md

# 次優先顯示
^nofa\/ji\/
參考資料\.md
```

#### 組間優先級

在設置頁面中，通過拖拽左側 `⋮⋮` 按鈕調整文件組的顯示順序（點擊 ↑/↓ 按鈕調整）。

#### 組內模式優先級

展開每個文件組，拖拽調整正則行的順序（點擊 ↑/↓ 按鈕調整），匹配順序會影響結果顯示。點擊「重置該組順序」可恢復為默認順序。

#### 保存與重置

- **💾 保存排序設置到文件組**：將當前的文件組間順序和組內模式順序寫入文件組的初始順序（會修改 `data.json` 中的模式排列）
- **🔄 重置所有排序設置**：清空所有排序配置，恢復默認狀態（文件組初始順序不變，需重新保存排序才能固化）
- **📁 管理文件組/組合**：快速跳轉到文件組管理界面

### 3. 歷史管理

| 設置項 | 說明 |
|:---|:---|
| 最大保存歷史數 | 設置歷史記錄上限（1-100），超出時自動截斷 |
| 歷史面板寬度 | 設置歷史列表彈窗寬度（200-600px） |
| 清除所有歷史 | 清空全部歷史記錄，重置當前索引為 -1 |
| 歷史列表 | 顯示所有歷史條目（搜索文本預覽 + 範圍顯示），支持單獨刪除 |

### 4. 顏色外觀

所有顏色均支持 RGBA 格式（紅、綠、藍、透明度），提供多種配置方式：

- **顏色選擇器**：直觀選取顏色（十六進制）
- **透明度滑塊**：調整透明程度（0-100%）
- **手動輸入框**：直接輸入 `rgba(r, g, b, a)` 或 `#RRGGBB` 格式，錯誤時恢復原值
- **實時預覽**：色塊實時顯示當前顏色
- **實時生效**：修改後立即刷新結果面板

#### 可自定義的顏色項

| 類別 | 顏色項 | 說明 |
|:---|:---|:---|
| 基礎顏色 | 文件名顏色 | 搜索結果中文件名的顏色 |
| | 匹配行背景色 | 匹配行的背景顏色（B/C 模式） |
| | 關鍵詞高亮背景色 | 搜索關鍵詞的高亮背景顏色 |
| | 懸停背景色 | 鼠標懸停時的行背景顏色 |
| 模式按鈕 | 模式 A/B/C 按鈕顏色 | 三種模式的按鈕背景色 |
| | 激活模式外框顏色 | 當前激活的全局模式按鈕的外框顏色 |
| 序號文字 | A/B/C 模式序號文字顏色 | 每種模式下序號數字的文字顏色 |

點擊「重置所有顏色為默認值」可恢復所有顏色設置。

## 💡 正則表達式提示

正則模式，帶標識：`* + ? $ { } \ [ ]` 

### 基本規則

| 規則 | 說明 |
|:---|:---|
| `\d` | 匹配數字 |
| `\d+` | 匹配一個或多個數字 |
| `\.` | 匹配點號（需要轉義） |
| `(A\|B)` | 匹配 A 或 B |
| `^` | 匹配開頭 |
| `$` | 匹配結尾 |
| `\/` | 匹配路徑分隔符（需要轉義） |

### 路徑匹配注意事項

在 Obsidian 中匹配路徑時，正則表達式中的 `/` 需要轉義為 `\/`：

```
✅ 正確：^kosa\/ju\/
❌ 錯誤：^kosa/ju/
```

### 排除規則

以 `!` 開頭的行表示排除匹配該模式的文件：

```
!.+疏22\.md        # 排除文件名包含「疏22」的文件
!^vibhasa\/others\/ # 排除 vibhasa/others/ 路徑下的文件
```

如需匹配以感嘆號開頭的模式，使用轉義 `\!`：
```
\!important        # 匹配以「!important」開頭的內容
```

### 布爾查詢模式

內容搜索中，**布爾查詢模式**（勾選「布爾查詢」後啟用）：

- 空格 = AND（與）
- `&` = AND（與）
- `|` = OR（或）
- `!` = NOT（非）
- `( )` = 分組

```
# 範例
佛性 如來藏           # 包含「佛性」且包含「如來藏」
佛性 | 如來藏         # 包含「佛性」或「如來藏」
佛性 !外道            # 包含「佛性」但不包含「外道」
(佛性|如來藏) 眾生    # 包含（佛性或如來藏）且包含「眾生」
```

### 註釋與空白

#### 文件名匹配
- 以 `#` 開頭的行會被忽略，可用於添加說明
- 行首的空格和製表符會被自動忽略（`trim()` 處理），如需匹配開頭空格和製表符請使用 `\s` `\t`

```
# 阿含部經典
(長阿含|中阿含|雜阿含|增一阿含)\.md

# 論藏文獻
(大毘婆沙|順正理論|俱舍論)\.md
```

#### 內容搜索

內容搜索中，開頭空格、`#`、`^` 等字符都可以直接正常使用，無需特殊處理。

**普通文本模式**（未勾選「布爾查詢」時）：

```
#普通文本
#定義區塊 ^block-id
#引用區塊 #^block-id
[[筆記#^block-id]]
[文字](網址#^block-id)
#標籤 #targ-id
```

**正則模式**（未勾選「布爾查詢」，且內容包含 `+*?${}()|[\]\\` 等符號時自動啟用）：

```
#定義區塊
\s\^block-id

#引用區塊
#\^block-id
\[\[.*?#\^block-id\]\]
\[[^\]]+\]\([^\)]+#\^block-id\)

#標籤
#targ-id
```
> ⚠️ 注意：布爾查詢模式下，`+*?${}()|[\]\\` 等符號視為普通字符，不會觸發正則模式。

## ❓ 常見問題

### Q: 搜索結果不顯示？
A: 確保文件名模式匹配到了文件，且文件中包含搜索關鍵詞。檢查正則語法是否正確。可查看控制台是否有錯誤信息。

### Q: 正則表達式報錯？
A: 檢查正則語法，注意路徑中的 `/` 需要轉義為 `\/`，特殊字符需要轉義。錯誤時會有通知提示具體原因。

### Q: 排序不生效？
A: 檢查「啟用結果排序」是否已開啟，且已配置相應的優先級規則。注意「文件優先級」規則優先級最高，會使匹配文件成為「優先文件」，無視後續排序規則（組間優先級、組內模式優先級只對普通文件生效）。

### Q: 如何恢復默認設置？
A: 在設置頁面「顏色外觀」標籤頁點擊「重置所有顏色為默認值」；在「優先級設置」標籤頁點擊「重置所有排序設置」；在「管理文件組/組合」中點擊「清除臨時默認」；在「搜索設置」中可手動重置默認文件模式。

### Q: 文件組/組合數據保存在哪裡？
A: 保存在 Obsidian 插件文件夾下的 `data.json` 中。插件版本更新時（版本不匹配）會自動備份舊配置（文件名：`data.json.backup.v{舊版本號}.{時間戳}.json`），並重置為默認配置，彈窗提示備份路徑。

### Q: 結果面板中的鍵盤導航不起作用？
A: 請確保先點擊結果面板區域（或按 Tab 鍵聚焦），使其獲得焦點，然後再使用方向鍵。檢查是否有其他元素搶佔了焦點。

### Q: 包含/排除規則如何工作？
A: 插件根據規則的出現順序自動判斷模式：僅包含規則 (`include-only`)、僅排除規則 (`exclude-only`)、先排除後包含 (`exclude-first`)、先包含後排除 (`exclude-last`)。如果同時存在包含和排除規則，會根據第一個包含行之後是否有排除行來決定順序。

> 💡 提示：排除行（以 `!` 開頭）建議統一放在所有包含行的**最前面**（先排除後包含）或**最後面**（先包含後排除），以確保過濾順序符合預期。混亂排列可能導致意外的過濾結果。

## 🔧 技術說明

- 變音符號忽略：將七組字符互轉為字符組正則，實現雙向匹配
- 布爾查詢：生成兼容的正則表達式
- 排除規則解析：支持三種排除（純包含、純排除、先排除後包含、先包含後排除），以 `!` 開頭的行為排除規則，`\!` 轉義為字面感嘆號
- 歷史字段自動修復：載入時檢查 `isBooleanQuery`、`enableDiacriticIgnore`、`rangeRef` 等字段，缺失則補充默認值，並修正布爾模式與正則模式的衝突
- 搜索結果緩存在右側邊欄視圖中（`SearchResultView`）
- 支持動態測量面板寬度，基於中文字符寬度自動調整單行模式的文本截取
- 使用 Obsidian 的 `MarkdownRenderer.renderMarkdown()` 渲染結果，支持格式保留
- 文件組/組合數據、搜索歷史保存在 `.obsidian/plugins/custom-search/data.json` 中
- 原生搜索使用 `executeCommandById("search:search")` 代替模擬 Enter 事件，避免 `isTrusted: false` 導致的問題
- 正則模式支持自動檢測路徑匹配（包含 `/` 或 `\/` 時轉為 `path:` 查詢），添加括號以確保原生語法解析正確
- 結果排序器（`ResultSorter`）支持三層優先級規則，只對普通文件應用 B/C 規則，優先文件（文件優先級匹配）不受後續排序規則影響
- 搜索歷史支持完全去重、最大數量限制、面板寬度可調、鍵盤導航
- 單行模式（A模式）自動轉義腳注行（以 `[^...]:` 開頭的行），自動刪除所有反引號以避免 Markdown 解析問題
- 文件組/組合編輯時自動調整文本域高度（自適應最大 300px）
- 懸浮模式按鈕使用延遲顯示/隱藏（200ms/250ms），避免誤觸發，背景為毛玻璃效果

### 二、搜索語法對比

| 運算符                             |         Custom-Search（本插件）         |      Obsidian 原生搜索      |
| ------------------------------- | :--------------------------------: | :---------------------: |
| **文件名/路徑 OR**（多選一）              |             ✅ 多行自動 OR              |  ✅ `file:A OR file:B`   |

|                                 |                                    |                         |
| 範圍類型                            |           Custom-Search            |          Obsidian 原生搜索           |
| **文件名正則**                       |               ✅ 完整支持               |    ⚠️ 僅 `file:/正則/`     |
| **路徑正則**                        |               ✅ 完整支持               |    ⚠️ 僅 `path:/正則/`     |
| **文件夾路徑**                       |           ✅ `^kosa/ju/`            |    ✅ `path:kosa/ju/`    |
| **文件組預設**                       |             ✅ 多組 OR 組合             |           ❌ 無           |
| **文件組組合**                       |              ✅ 組間可再組合              |           ❌ 無           |
